最短路

最短路是图论中最基本的一类问题。可以分为单源最短路和多源最短路两类。

单源最短路

Dijkstra

最高效的最短路算法，但是不能处理有负边权的图。
其本质是不断贪心。

考虑以某一点u为起点的最短路。

检查从这一点的所有出边，设短的那条出边为e。显然到e的终点（设为v）的最短路已经确定，长度就是e的长度w。因为若有其他的最短路，其必然包含不是e的其他出边，又因为因为e是最短的出边，所以包含非e边的这条路径长度就已经大于w了。

然后扫描v的所有出边，再次找到从u出发到一个没有确定最短路的点（设为v1）的最短路径。那么同上到v1的最短路径长度也被确定了。

继续上面的步骤直到所有的点的最短路确定。

具体实现中，可以加入一个堆（优先队列）来维护最短路程到达的点。这样做的话时间复杂度为O(nlogn)

因为负边权会影响贪心。
所以这个算法不能解决负边权最短路问题。

int dis[N],n,s;
bool book[N];
void dijkstra(){
	for(int i=1;i<=n;i++)dis[i]=INF;
	priority_queue<node> q;
	memset(book,0,sizeof(book));
	dis[s]=0;
	q.push(node(s,0));
	while(!q.empty()){
		int u=q.top().x;
		q.pop();
		if(book[u]==1)continue;
		book[u]=1;
		for(int i=head[u];i;i=e[i].nxt){
			int v=e[i].v,w=e[i].w;
			if(dis[v]<=dis[u]+w)continue;
			dis[v]=dis[u]+w;
			q.push(node(v,dis[v]));
		}
	}
}

Bellman-Ford

此算法基于一个基本操作——松弛操作

即 if(dis[v]>dis[u]+w)dis[v]=dis[u]+w（dis[u]代表目前到u点最短路的最优解，w是边的权）

显然如果对整个边集做一次松弛操作就会使答案更优，当答案不能更优的时候，dis即为最短路。

考虑时间复杂度

考虑每次松弛操作之后会有几个点的最短路成为最优解？

通过讲解Dijkstra可以知道，从一个点集松弛一次至少可以得到一个点的最短路的最优解。

那么最差的情况呢？

如图，按图中边的顺序松弛，要松弛5次才能获得所有最短路。

所以不加证明地得出：从一个点集松弛一次最差只能得到一个点的最短路的最优解。
严格证明传送门

所以复杂度为O(mn)

spfa

此为以上算法优化，因为只有松弛成功地v才有意义，所以用一个队列装进所有有意义的v点，以此取出队列中的点对其所有边松弛即可。

对于Bellman-Ford中的最差情况有较好的效率。

spfa可以处理负边权（其本质就是优美的暴力），可以查负环。一度被人们（包括我）认为是最好的的最短路算法。

然而人们（包括我）使用它时都遗忘了对其复杂度的精确证明（就跟使用dinic时一样）。这是一个隐患，但是人们并不在意。终于有一天NOI2018归程出题人精心设计的数据把无数人卡了（包括我学长）。人们才给它判了死刑。

为什么这么多人喜欢spfa？因为spfa在跑随机图时确实非常给力。
主要原因还是spfa好写(笑)

#include<iostream>
#include<cstring>
#include<cmath>
#include<cstdio>
#include<algorithm>
#include<queue>
using namespace std;
#define N 10010
#define M 500010
#define INF 2147483647
struct edge{
	int nxt,v,w;
}e[M];
int cnt,head[N];
void add_edge(int u,int v,int w){
	cnt++;
	e[cnt].v=v;
	e[cnt].w=w;
	e[cnt].nxt=head[u];
	head[u]=cnt;
}
int dis[N],num[N],n,s;
bool vis[N];
void spfa(){
	for(int i=1;i<=n;i++)dis[i]=INF;
	queue<int> q;
	memset(vis,0,sizeof(vis));
	vis[s]=1;
	q.push(s);
	dis[s]=0;
	num[s]=1;
	while(!q.empty()){
		int u=q.front();
		q.pop();
		vis[u]=0;
		for(int i=head[u];i;i=e[i].nxt){
			int v=e[i].v,w=e[i].w;
			if(dis[v]<=dis[u]+w)continue;
			dis[v]=dis[u]+w;
			num[v]=num[u]+1;
			if(vis[v]==0){
				q.push(v);
				vis[v]=1; 
			}
			if(num[v]>n)return;
		}
	}
}

多源最短路

n次单源最短路

不多说了。

Floyd

$O(n^3)$的算法，大多数时候优于$n$次$dijkstra$。but它不能处理负环。

枚举一个中间点$k$，然后枚举点对$(i,j)$。尝试用$dis[i,k]+dis[j,k]$松弛$dis[i,j]$。

为什么这样时可行的？更换枚举顺序行不行（先枚举点对再枚举中间点）？

考虑任意一条最短路。最短路上的所有点都会作为k被枚举，然后联通相邻点集的最短路。最后这条路整个会被联通。

如图，方块中是已经确定彼此最短路的点集（因为最短路总是有一个个最短路组成，所以一条最短路上点彼此之间的最短路是整个最短路的一部分），枚举k为中间点之后，i会包含进来，显然最后所有点都会包含进来。

先枚举点对再枚举中间点毫无道理，所以会出问题。

代码太简单就不写了。

Johnson

有的时候n次dijkstra比Floyd更加高效（比如比较稀疏的图），但是苦于能解负边权的问题。其实dijkstra是可以处理带负边权的图的，只是需要一些技巧。

（是的，johnson算法提出时是为了解决稀疏图中的多源最短路问题，但是它最重要的思想是提出了解决dijkstra不能解决负边权问题的解决方法）

一个比较自然的想法是把所有边权加上一个x使其全部大于0。但是这显然错误:
首先是因为有负环的存在。这种情况下有的点对之间根本没有最短路。
其次在每一个边上加一个数会使比较短（这里指包含边数较少）的路更优。

正确的思路是新建一个节点s，用边权为0的有向边连接s和其他点。然后以s为起点跑一个Bellman-Ford或者spfa（可以处理负边权或负环的算法）（这也是johnson为什么是多源最短路算法的原因。如果是单源最短路，既然要跑一边spfa再跑dijkstra，为什么不直接跑spfa），得到从s到每一个点的最短路，记为f[i]。然后把原图中所有边权换成w’=w+f[u]-f[v]。然后跑dijkstra就是正确的。

考虑其正确性。主要从两个方面考虑：1、在新图中跑出来的最短路是否是原图的最短路。2、新图中所有边的边权是否非负。

第二点比较好证明：在跑第一次spfa之后必然有不等式f[v]<=f[u]+w。w’=w+f[u]-f[v]>=0。

然后是第一点：使用势能证明(似乎splay也是用势能证明的)。势能的特点是，两个点势能之差一定是一个常数，跟零势能处的位置无关。然后考虑新图中一条路的长度：w1’+w2’+w3’+···+wm’=(w1+f[s]-f[p2])+(w2+f[p2]-f[p3])+(w3+f[p3]-f[p4])+···+(wn+f[pn]-f[t])=w1+w2+w3+···+wn+f[s]-f[t]（pn代表这条有向道路的第n个点，s是起点，t是终点）。可以发现新图中从s到t的任意道路长度由两部分组成，一个是原始道路长度，另一个是一个只和s，t有关的量（就像势能一样）。其路径的长短只和原始道路长短有关。故新图中的最短路就是原图中的最短路。

#include<iostream>
#include<cstring>
#include<cstdio>
#include<cmath>
#include<algorithm>
#include<queue>
using namespace std;
#define N 3010
#define M N+6010
#define INF 1e9
#define int long long
struct edge{
	int nxt,v,w;
}e[M];
int cnt,head[N];
void add_edge(int u,int v,int w){
	cnt++;
	e[cnt].nxt=head[u];
	e[cnt].v=v;
	e[cnt].w=w;
	head[u]=cnt;
}
int f[N],num[N],n,s;
bool vis[N];
void spfa(){
	for(int i=1;i<=n;i++)f[i]=INF;
	memset(vis,0,sizeof(vis));
	queue<int> q;
	q.push(s);
	vis[s]=1;
	f[s]=0;
	num[s]=0;
	while(!q.empty()){
		int u=q.front();q.pop();
		vis[u]=0;
		for(int i=head[u];i;i=e[i].nxt){
			int v=e[i].v,w=e[i].w;
			if(f[v]<=f[u]+w)continue;
			f[v]=f[u]+w;
			num[v]=num[u]+1;
			if(num[v]>n)return;
			if(vis[v])continue;
			q.push(v);
			vis[v]=1;
		}
	}
}
struct node{
	int x,s;
	node(int X,int S){
		x=X,s=S;
	}
};
bool operator <(node a,node b){
	return a.s>b.s;
} 
int dis[N];
void dijkstra(int s){
	for(int i=1;i<=n;i++)dis[i]=INF;
	memset(vis,0,sizeof(vis));
	dis[s]=0;
	priority_queue<node> q;
	q.push(node(s,0));
	while(!q.empty()){
		int u=q.top().x;
		q.pop();
		if(vis[u])continue;
		vis[u]=1;
		for(int i=head[u];i;i=e[i].nxt){
			int v=e[i].v,w=e[i].w;
			if(dis[v]<=dis[u]+w)continue;
			dis[v]=dis[u]+w;
			q.push(node(v,dis[v])); 
		}
	}
}
int read(){
	int sum=0,f=1;
	char ch=getchar();
	while(ch<'0'||ch>'9'){f=-1;ch=getchar();}
	while(ch<='9'&&ch>='0'){sum=sum*10+ch-'0';ch=getchar();}
	return sum*f;
}

	n=read(),m=read();
	for(int i=1;i<=m;i++){
		int u=read(),v=read(),w=read();
		add_edge(u,v,w);
	}
	s=0;
	for(int i=1;i<=n;i++)add_edge(s,i,0);
	spfa();
	for(int i=1;i<=n;i++)
		if(num[i]>n){
			printf("-1");
			return 0;
		}
	for(int i=1;i<=n;i++)
		for(int j=head[i];j;j=e[j].nxt)e[j].w=e[j].w+f[i]-f[e[j].v];
	for(int i=1;i<=n;i++){
		dijkstra(i);
		int ans=0;
		for(int j=1;j<=n;j++)ans+=j*min((int)INF,dis[j]+f[j]-f[i]);
		printf("%lld\n",ans);
	}
	return 0;
}
