指针学习笔记，不仅仅是指针

发表于 2020-12-17- 更新于 2020-12-20

我一个学过C语言的居然不会指针

什么是指针？

指针是$C$语言中的一种特殊的数据类型。指针就是用来存储地址的变量。

指针有什么用？

目前我认可的优势有以下几点：
1）指针的使用使得不同区域的代码可以轻易的共享内存数据，这样可以使程序更为快速高效。
2）C语言是传值调用，而有些操作传值调用是无法完成的，如通过被调函数修改调用函数的对象，但是这种操作可以由指针来完成，而且并不违背传值调用。

预备知识

关于地址

为了更好地理解指针。最后好要明白数据是怎样在内存中储存的。
（以下以个人计算机为例）
计算机中CPU可以直接访问的、用来存储数据的记忆部件成为存储器。存储器又由无数个存储单元（其中存储单元就是一个字节）组成。每一个单元由一个唯一的编号表示，这个编号就是这个单元的内存地址。
可以用下图形象的理解：

本图为一个4G的内存，每个单元的地址由左侧的16进制数表示，而单元中的数据则像数组一般整齐地排列着。

直接寻址&间接寻址

直接寻址即通过变量名存取变量内容的访问方式。
当你定义一个$int$型变量（4个字节）时计算机就会选择连续的4个存储单元并修改其内容使其正确表示此$int$型变量。在程序中一般是通过变量名对内存单元进行存取操作的。其实程序经过编译以后已经将变量名转换为变量的地址（变量的地址就是其对应的若干个存储单元的第一个单元），对变量值的存取是通过地址进行的。例如语句

printf("%d",n);

是这样执行的：根据变量名与地址的对应关系，找到变量i的地址如$0x000000F5$，然后从由$0x000000F5$开始的4个字节中取出数据。

间接访问即想访问指针变量获得变量的地址，再到相应的地址去访问变量。

两种运算&和*

&变量名
获取变量的首个字节地址。
*指针名
获取一个指针变量储存地址的内容。可以当做一个变量来用。
下面用$C++$的语法演示(本文代码若无特殊说明皆为$C++$语法)

int x=3,y=4;
p=&x;//p为一个整形指针变量
printf("%d\n",*p+y);
*p+=3;
printf("%d",*p);

指针的定义

如前所述，指针是一种存放地址的变量，像其他的变量一样，必须在使用前定义。指针变量的命名遵守与其他变量相同的命名规则。即必须是唯一标识符。
指针定义的格式如下：
类型名 *变量名
其中类型名说明指针所指向变量的类型。星号$$是一个指针变量的标志，说明指针名是一个“类型名”类型的指针而不是一个“类型名”类型的变量。指针可以与非指针变量一起说明。举几个例子：

int *p,a;

定义了一个整型指针$p$和一个*整型变量a
int * p,a;

与上第一个例子等价。
$ps$：指针指向一个变量意思是指针存储着此变量的地址。

一个疑问

Q：指针不是储存地址的变量吗，为什么还要定义时还要定义一个“变量名”类型？
A：指针确实只是储存地址，但是刚才提到$^*p$时计算机会获取以$p$中地址为开始的若干个存储单元的内容。而$char$类型变量只占一个字节，而$int$就占4个字节。计算机怎么会知道取几个存储单元？所以就需要“变量名”类型告诉计算机。

指针初始化的必要性和NULL指针

一个没有初始化的指针的值是随机的，也就是说它可以指向内存的任何位置，甚至是操作系统占用的内存位置，所以修改一个没有初始化的指针是很危险的。

NULL 指针是一个特殊的指针变量，表示不指向任何东西。可以通过给一个指针赋一个零值来生成一个 NULL 指针。

int *p;
p=NULL; 
printf("%p",p);

可以看到指针的地址为0。在大多数的操作系统上，程序不允许访问地址为 0 的内存，因为该内存是为操作系统保留的。但是，内存地址 0 有一个特别重要的意义，它表明该指针不指向一个可访问的内存位置。

学了指针之后做的第一件事

试试利用按地址调用写一个$swap$(交换)函数。
虽然$C++$的库中有这个函数，但是不得不承认$swap$确实是一个按地址调用好例子。所以本文还是以$swap$为例。

void swap(int *a,int *b){
	int c=*a;*a=*b;*b=c;
	return;
}
int main(){
	int x=1,y=2;
	swap(&x,&y);
	return 0;
}

这样经过$swap$后$x$和$y$的值会对调。为什么呢？

程序中把$x$与$y$的地址分别付给了指针变量$a$和$b$此时$^a$和$^b$的修改就不是普普通通的两个变量的修改。
刚才提到$^p$时计算机会获取以$p$中地址为开始的若干个存储单元的内容，我们对$^a$和$^*b$的修改其实就是对指针变量$a$和$b$对应地址的变量修改，即对$x$与$y$修改。

然而我仍不知道以前写的与此例有相同效果的

void swap(int &a,int &b){
	int c=a;a=b;b=c;
	return;
}
int main(){
	int x=1,y=2;
	swap(x,y);
	return 0;
}

的原理。。。。(划掉)

学会指针后做的的第二件事。

指向指针类型的指针。

int *p;
int **q;//定义一个指向指针的指针
int x=3;
p=&x;
q=&p;
printf("%d\n",*q);//ppx的内容作为地址的内容->p
printf("%d\n",**q);//ppx的内容作为地址的内容再作为地址的内容->x

指针的运算

1）指针和整型的加减。
如定义一个$int ^*p$，$p+1$即$p$存储的地址加4（指针类型字节数）之后的地址。减法与加法相似。
2）指针 - 指针
　　只有当两个指针都指向同一个数组中的元素时，才允许从一个指针减去另一个指针。两个指针相减的结果的类型是 $ptrdiff_t$，它是一种有符号整数类型。减法运算的值是两个指针在内存中的距离（以数组元素的长度为单位，而不是以字节为单位），因为减法运算的结果将除以数组元素类型的长度。

再次理解数组

数组就是把多个相同类型的变量紧凑地放置到一起。
对于一个数组$a[]$，$a$即数组的首地址（第一个元素的地址），$a+1$为数组第二个元素的地址……
仔细发现，大胆猜想我们可以发现其实$a$就相当于一个指针！！！
$a[i]$和$^*(a+i+1)$是等价的

指针数组和数组指针

指针数组

指针变量可以同其他变量一样作为数组的元素，如果一个数组的元素是由指针变量组成，那么这个数组成为指针数组。
指针数组的定义形式为：
类型名 *数组名[常量表达式]
为什么要定义和使用指针数组呢？主要是由于指针数组对处理字符串提供了更大的方便和灵活，使用二维数组在处理长度不等的正文时效率较低，而指针数组由于其中每个元素均为指针，因此处理起来更为方便。

数组指针

数组指针是指向一个数组的指针，$p+1$会跨过n个变量，所以其应用与高维数组有关。
用例子理解：

int (*p)[10];//p指向一个一维数组
int a[5][10];
p=a;//p的值为a的首地址。值得注意的是及时a与a[0]在数值上是相等的，但是这一句改成p=a[0]会报错。p是指向一个一维数组的指针，a可以理解为元素为一维数组的一维数组的首元素地址，两个都是地址逻辑上与p对等。而a[0]则是一个a这个以一维数组为元素的一维数组的第一个元素，一个元素一个地址与p指针逻辑不对等。
a[0][1]=1;a[1][2]=3;
printf("%d\n",(*p)[1]);//a[0][1];
p++;
printf("%d",(*p)[2]);//a[1][2];
//也可以写成*(*p+2);

类似的：
int (*p)[10][10];
int a[5][10][10];
p=a;
a[0][1][1]=1;a[1][2][2]=3;
printf("%d\n",(*p)[1][1]);//a[0][1][1];
p++;
printf("%d",(*p)[2][2]);//a[1][2][2];

动态分配内存——动态数组

久仰大名，今天前来学习。
变量定义后编译器给变量分配相应的内存，这种变量的内存分配是在程序运行一开始就确定下来，而且所分配内存的大小也随同变量类型被固定下来，不能再变化。
如果一个题目中给定最多10个个班级，每班有至多300个人。可以定义一个$a[10][300]$的数组，但是往往会浪费很多空间。
最理想的解决方案就是一个在运行时才确定大小的数组及动态数组。
$C$语言中动态分配内存的函数主要有两个：$malloc()$（内存分配，memory allocation）和$calloc()$（连续内存分配 contiguous allocation）,两个函数在$algorithm$和$stdlib.h$（在C语言则是$stdlib.h$)
其原理是计算机在运行时，除了被系统和用户使用的内存之外，还有一部分未使用的内存空间——堆（heap），相信很多人看到堆会会心一笑，但是这个堆并不是那个数据结构。$malloc$和$calloc$可以从堆里获得内存。堆的大小是可以动态改变，如果堆的内存不足，系统就会在空闲内存中继续分配内存给堆，直到空闲内存被用完。

malloc

其函数原型为：

void *malloc(unsigned int num_bytes);

唯一的参数是需要分配的字节数。不会自动初始化

calloc

其函数原型为

void *calloc(unsigned n,unsigned size);

两个参数第一个是分配多少个对应变量的内存，第二个是对应变量的字节数。会自动初始化成该类型的缺省值，数值型为0，字符型为’\0’。

我发现了

本函数的返回值是$void ^*$
这表示该函数返回一个通用类型的指针，在实际使用中需要把这个通用类型的指针转化成一种特定类型的指针使用。
PS：动态分配的内存是没有名字的，需要你给他名字。
举个实际的栗子：

int *t;
t=(int *)malloc(7*sizeof(int));
或
t=(int *)calloc(7,sizeof(int));

free

函数原型：

void free(void *FirstByte);

$free$只有一个参数就是$malloc$或$calloc$返回内存块的首地址。如果不是由内存分配函数返回的首地址则可能出现无法预知的问题。
PS:
如果没有足够的内存，则函数$malloc$和$calloc$会得到$NULL$的返回值。
这时一般用
if(!t)exit(1);

来建立错误陷阱.
$PS:$
$exit$为$C++$的退出函数，声明于$stdlib.h$中，对于$C++$其标准的头文件为$cstdlib$,声明为
$void exit(int value);$
$exit$的功能为，退出当前运行的程序，并将参数$value$返回给主调进程。
在$main$中$return v;$的效果 与$exit(v);$相同。
$exit(1)$和$exit(-1)$
是分别返回1和-1到主调程序。
$exit(0)$则是返回0。
$exit(0)$表示程序正常退出，非0表示非正常退出

例子

int *p;
p=(int *)calloc(10,sizeof(int));
if(p==0){
	printf("没有足够内存");
	exit(1); 
}
for(int i=1;i<=5;i++){
	p[i]=i;
	printf("%d\n",p[i]);
}
free(p);

指针与结构体

比较简单，举个例子：

struct tree{
	int ls,rs,sum;
}tr;
int main(){
	tr={1,2,3};
	tree *p=&tr;
	printf("%d %d",*p->ls+*p->rs,*p->sum);
	return 0;
}
